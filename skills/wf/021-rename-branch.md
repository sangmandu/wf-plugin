# Step 021: RENAME_BRANCH

## Purpose

The worktree was created before the Linear ticket, so the branch starts with a placeholder name generated by Claude Code (e.g. `worktree-serialized-fluttering-seal`). After `LINEAR_TICKET` runs, rename the branch to the ticket ID so all subsequent steps (commit, PR, CI) operate on the canonical name.

## Checklist

- [ ] Read `branch_name` and `ticket_id` from `state.json`.
  - [ ] If `ticket_id` is empty (e.g. `light` track skipped ticket creation for a docs/config-only change), mark this step completed and move on — keep the placeholder branch name as-is.
  - [ ] If `branch_name` already matches `ticket_id`, mark this step completed and move on — there is nothing to rename.
- [ ] Rename the local branch:
  ```bash
  git branch -m <old_branch_name> <ticket_id>
  ```
- [ ] Persist the new branch name to `state.json` by updating `branch_name`.
- [ ] Verify `git status` is clean and `git branch --show-current` returns the ticket ID.

Per `helpers#state_transition` — complete `RENAME_BRANCH`
